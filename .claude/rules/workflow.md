# Workflow Rules

- Always ask before: deleting files, removing dependencies, dropping DB columns/tables, overwriting existing code
- Only change what's necessary — no unsolicited refactoring
- Check existing conventions and patterns before introducing new ones
- Never edit files without reading them first
- Before installing packages, check if functionality already exists in dependencies
- Provide brief summary of all files changed after multi-file tasks
- Ask if identical components should be reused across pages rather than duplicated
- Kill development servers after testing
