# Naming Conventions

- All variables and identifiers in English only
- Consider industry vocabulary when naming — ask if unsure
- kebab-case for URL routes and frontend file names
- snake_case for DB columns, tables, Python functions and variables
- lowerCamelCase for TypeScript/JavaScript code
- UPPER_SNAKE_CASE for environment variables (`NEXT_PUBLIC_` prefix for client-visible)
- Singular table names for entities
- `_at` suffix for timestamps (`created_at`, `updated_at`)
- `_id` suffix for foreign keys (`user_id`, `organization_id`)
