# Environment Setup Guide

> How to configure environment variables for each deployment stage.

## Backend (`backend/.env`)

| Variable | Required | Dev default | Staging/Prod | Notes |
|---|---|---|---|---|
| `APP_ENV` | Yes | `development` | `staging` / `production` | Controls debug features, docs visibility |
| `APP_DEBUG` | No | `false` | `false` | Never `true` in production |
| `DATABASE_URL` | Yes | `postgresql+asyncpg://...@localhost:5432/aipoweredmakers` | Managed PostgreSQL URL | Must use `asyncpg` driver |
| `SECRET_KEY` | Yes | Generated by `make setup` | Unique per environment | Min 32 chars, `secrets.token_urlsafe(32)` |
| `LOG_LEVEL` | No | `DEBUG` | `INFO` | `DEBUG` / `INFO` / `WARNING` / `ERROR` |
| `CORS_ORIGINS` | No | localhost:3000–3015 (auto) | Explicit comma-separated URLs | **Must** set explicitly in production |
| `RATE_LIMIT_DEFAULT` | No | `100/minute` | `100/minute` | Per IP. Format: `N/second`, `N/minute`, `N/hour` |
| `REDIS_URL` | No | — | `redis://host:6379/0` | Optional. Enable with `uv add aipoweredmakers-backend[redis]` |
| `SENTRY_DSN` | No | — | Project DSN from Sentry | App works without it |
| `SENTRY_TRACES_SAMPLE_RATE` | No | `0.0` | `0.1`–`0.2` | `1.0` for local debugging. Keep low in prod to manage costs |
| `SENTRY_ENVIRONMENT` | No | — | `staging` / `production` | Falls back to `APP_ENV` if unset |

## Frontend (`frontend/.env.local`)

| Variable | Required | Dev default | Staging/Prod | Notes |
|---|---|---|---|---|
| `NEXT_PUBLIC_API_URL` | Yes | `http://localhost:8000` | Production API URL | Inlined at build time in production |
| `NEXT_PUBLIC_SENTRY_DSN` | No | — | Project DSN from Sentry | App works without it |
| `NEXT_PUBLIC_SENTRY_ENVIRONMENT` | No | — | `staging` / `production` | |
| `NEXT_PUBLIC_SENTRY_TRACES_SAMPLE_RATE` | No | `0` | `0.1`–`0.2` | `1` for local debugging |

## CORS Configuration

- **Local dev:** No config needed — defaults cover `localhost:3000`–`3015`
- **Production:** Set `CORS_ORIGINS` to the exact frontend URL(s), comma-separated
- `NEXT_PUBLIC_API_URL` in dev mode works dynamically; production builds inline it at build time

## Rate Limiting Behind a Reverse Proxy

slowapi uses `request.client.host` for rate limiting by IP. Behind a reverse proxy (Coolify/Traefik):

1. Configure the proxy to set `X-Forwarded-For` headers
2. Ensure only trusted proxies can set this header (prevent spoofing)
3. FastAPI will pick up the real client IP via the proxy headers

If rate limiting targets the proxy IP instead of client IPs, review your proxy's `X-Forwarded-For` configuration.

## Sentry Configuration

- **DSN only (no sample rate):** Safe — traces default to `0.0`, so no performance data is sent
- **Local debugging:** Set sample rate to `1.0` (backend) / `1` (frontend) temporarily
- **Production:** `0.1`–`0.2` balances visibility and cost
- Backend and frontend use separate Sentry projects/DSNs
